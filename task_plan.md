# Data Comparison Research Plan

## Goal

评估当前多智能体态势感知平台的数据展示能力，找出适合论文展示的更完整对照数据，并给出可执行的增强方向。

## Phases

| Phase | Status | Description |
|---|---|---|
| 1 | complete | 盘点前端页面、后端接口和现有输出数据 |
| 2 | complete | 识别当前数据展示缺口和论文展示痛点 |
| 3 | complete | 设计 2-3 组更完整的数据对照方案 |
| 4 | complete | 输出推荐方案和后续实现建议 |
| 5 | complete | 完成后端统计增强与收益代价增强实现 |
| 6 | complete | 前端接入新增实验数据并增强可视化展示 |

## Notes

- 本轮先做规划与调研，不直接修改业务代码。
- 重点关注实验对照是否足够支撑论文中的“性能分析”和“方法比较”。
- 当前仓库与用户截图中的大屏前端不是同一套源码，应以仓库真实数据链路为准给建议。
- 结果目录存在多批次实验文件混杂的问题，统一权威数据批次应作为后续实现的前置步骤。
